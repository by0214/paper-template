\section{Analysis}
\label{sec:analysis}
In the main text, we retain only analyses that directly support our core
claims, namely how SPRM constructs reliable local credit from outcome-only
trajectories and how the global prior complements that local signal.
Implementation-focused analyses, including Stage 1 model selection, input
feature selection, and IG baseline selection, are deferred to
Appendix~\ref{sec:appendix}.

\subsection{Ablation Study}
\label{ablation_study}
Under fixed backbone, training-data scale, and optimization hyperparameters, we
perform targeted ablations over the key SPRM mechanisms.

Table~\ref{tab:ablation_local_consistency} reorganizes the ablation around two
questions: how to construct the local credit signal itself, and how to make
that signal reliable enough to supervise the final PRM.

Panel A first compares three ways of forming local credit.
Removing the \texttt{question\_end} representation causes a clear collapse
across all metrics, confirming that local attribution should not start from the
first reasoning step alone.
The transition from the problem statement to the reasoning chain already
contains information about how subsequent steps should be interpreted, so
including \(x_{i,0}\) gives the local scorer a better boundary anchor.
The ordered value difference \(\Delta v\) remains a useful deterministic
reference, but the final method is built on IG-based attribution because it can
be paired with an explicit consistency mechanism rather than used as a fixed
prefix difference alone.

Panel B then asks how to denoise that IG-based local signal.
Both consistency mechanisms substantially improve over using raw IG signs
directly, which confirms that local credit needs an explicit reliability check
before it is used as supervision.
The LLM-based consistency signal yields the highest average F1, mainly through
stronger error rejection, while \(R_{i,t}^{\mathrm{causal}}\)-based consistency
preserves the best correct-path acceptance.
We adopt \(R_{i,t}^{\mathrm{causal}}\) in the final method because it is more
faithful to the quantity defined in \Cref{sec:method_local}: whether the
current step changes the predicted value of the particular trajectory in which
it appears.
